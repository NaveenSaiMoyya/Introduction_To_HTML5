Request/Response CYCLE:


                         http://si.michd.edu   ->  URL (address)

Client:
       - machines for personal use (laptop, phone)

Server:
       - machines holding shared resourses always connected to network.

Networks:
          LAN - Local Area Network (buildingd)
          WAN - Wide Area Network  (university)
    largest WAN - INTERNET

--> CLIENT requests for a webpage & SERVER must respond with appropriate files.

URL:
     - Uniform Resourse Locater
     - has 3 parts -_ _ Protocol
                   -_
                     -_ domain
                   -_
                     -_ document


PROTOCOLS:
           - HyperText Transfer Protocol (HTTP)
           - Secure HTTP (HTTPS)
           - File Transfer Protocol (FTP)

DOMAIN:
          - determined by ICANN (Internet Corporation for Assigned Names and Numbers)
          - .edu, .org, .gov etc..
          - domain names are linked to IP Address
          - before domain names, used to type numbers to access webpages


IPV6:
         - Internet Protocol Version 6
         - communication protocol that identifies computers on network
         - every computer has a unique IP Address

DOCUMENT:
         - Every server will be having its default document.
         - By convention, "index.html" is default.


REQUEST:
         - Browser creates an HTTP request.
         - contains 'header', 'cookies', 'form data', etc..


RESPONSE:
         - Server returns files & browser decides what to do with files.
         - usually browser turns the files into webpages.
         - If server can't fulfill the request, it sends back files with error codes.

404 ERROR: File Not Found
500 ERROR: Server is Down
